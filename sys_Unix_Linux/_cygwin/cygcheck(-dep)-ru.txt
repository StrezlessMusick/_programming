cygcheck - выводит системную информацию, проверяет установленные пакеты, или зпросить базу данных пакетов
>> query - запрос

SYNOPSIS
       cygcheck [-v] [-h] PROGRAM
       cygcheck -c [-d] [PACKAGE]
       cygcheck -s [-r] [-v] [-h]
       cygcheck -k
       cygcheck -f FILE [FILE]...
       cygcheck -l [PACKAGE]...
       cygcheck -p REGEXP
       cygcheck --delete-orphaned-installation-keys
       cygcheck --enable-unique-object-names Cygwin-DLL
       cygcheck --disable-unique-object-names Cygwin-DLL
       cygcheck --show-unique-object-names Cygwin-DLL
       cygcheck -h

>> at least - по крайней мере
	требуется по крайней мере одна опция или PROGRAM, как показано выше
	PROGRAM
		перечисляет библиотеки (DLL), от которых зависит программма
	-c, --check-setup
		показывает установленную версию пакета PACKAGE и проверяет его целостность (или для всех установленных пакетов, если ни одн не задан)
>> integrity - целостность
	-d, --dump-only
		только перечисляет пакеты, без проверки (мвесте с -c)
	-s, --sysinfo
		производит (produce) диагностическую системную информацию (включая -c -d)
	-r, --registry
		также сканирует реестр для настроек Cygwin (вместе с -s)
	-k, --key-check
		производит сессию настройки клавиатуры (должен быть запущен только из консоли, не из pty/rxvt/xterm)
	-f, --find-package
		находит пакет, которому принадлежит FILE

cygcheck-dep - показывает информацию о зависимостях для установленных пакетов Cygwin

Usage: cygcheck-dep [-v] [-c] [-s FILE | -S] [-p] [-l] [-i | -I]
       cygcheck-dep [-v] [-c] [-s FILE | -S] [-p] [[-r] [-R] [-n] [-N] [PACKAGE ...]]
       cygcheck-dep [-v] [-c] [-s FILE | -S] [-o] [-b] [-f]
       cygcheck-dep [-h | --help]
       cygcheck-dep [-V | --version]

Опции:
  (диагностика установленного)
	-l	Найти установленные пакеты, которые не требуются ни для каких други пакетов (которые "сбежали" из дерева зависимостей пакетов)
	-i	Найти "острова" пакетов. "Остров" - это группа взаимозависимых пакетов, которые требуются друг для друга (с циклическими или все для всех зависимостями) 
			и любая часть или все вместе они не требуются ни для каких других установленных пакетов.
	-I	показать группы пакетов с циклически или все от всех зависимостями. Заменяет -i.
>> supersede - заменять
	-b	Показать список установленных пакетов, которые имеют сломанные или неизвестные зависимости. Включает -q.
	-f	Показывает список установленных пакетов, которые имеют фантомные зависимости (несуществующие пакеты, такие как переименованные, удаленные, и т.д.), указанные в файле setup.ini. Включает -q.
>> retire - уходить в отставку
>> tire - шина, покрышка, бондаж; утомлять, уставать...
  (Информация о пакетах)
	-r	Показать пакеты, которые требуются для PACKAGE. Это тот же список, который задан в строке "requires:" в файле setup.ini для PACKAGE, за исключением сломанных и фантомных зависимостей.
	-R	Рекурсивно разворачивает список пакетов, котрые требуются для PACKAGE.
	-n	Показывает пакеты, которые нуждаются в PACKAGE
	-N	Рекурсивно разворачивает список пакетов, которые нуждаются в PACKAGE.
  (поддержка портов Cygwin)
	-p	Включает поддержку коллекции портов Cygwin.
	-o	Перечисляет пакеты, которые имеют переопределенные версии в коллекции портов Cygwin. Включает -q -p.
>> override - отменять, отвергать...
>> overridden - переопределенный
  (Модификаторы)
	-c	Обычно cygcheck-dep запускает скачивание последнего файла базы данных пакетов Cygwin (setup.ini) с информацией о зависимостях пакетов с ftp сайта Cygwin.
		Эта опция предупреждает необязательное скачивание, и заставляет использовать файл, запомненный при предыдущем скачивании.

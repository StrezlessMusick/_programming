file - определяет тип файла 
     file [-bcEhiklLNnprsvz0] [--apple] [--mime-encoding] [--mime-type] [-e testname] [-F separator] [-f namefile] [-m magicfiles] file ...
     file -C [-m magicfiles]
     file [--help]

>> attempt - попытка
file тестирует каждый аргумент в попытках классифицировать его. Усть три набора тестов, выполняемые в следующем порядке: тесты файловой системы, магические тесты и языковые тесты. Первый тест, который пройден вызывает печать типа файла.

Печатаемый текст обычно содержит одно из слов text (файл содержит только печатаемые символы и несного общих управляющих симовлов, и возможно является безопасным для чтения на ASCII терминале), executable (файл содержит результат вычисления программы в форме понятной для некоторого ядра UNIX или другого) или data означает что-либо еще (данные обычно "бинарные", непечатаемые). ...

Тесты файловой системы обычно базируются на анализе результат системного вызова stat(2). Программа смотрит, пуст ли файл, или это какой-то специальный тип файла. 
>> appropriate - соответствующий.
.... определены в системном заголовочном файле <sys/stat.h>.

----------------- --help
usage: file [OPTION...] [FILE...]
определяет тип файлов

	--help		показывает этот хелп и выходит
  -v	--version	выводит информацию о версии и выходит
  -d	--debug		печатает отладочные сообщения

  -m	--magic-file LIST использует LIST как разделенный двоеточиями список магических чисел/номеров файлов
>> colon - двоеточие
  -F	--separator STRING использует строку как сепаратор вместо ':'
  -c	--checking-printout печатает разобранную форму магического файла, используемого в соответствии с -m
			для отладки нового магического файла перед установкой его
  -l	--list		перечисляет магическую силу
  -C	--compile	компилировать файл, заданый -m

  -f	--files-from FILE читает имена файлов для проверки из FILE'а
  -z	--uncompress	пытается заглядывать внутрь сжатых файлов
  -L	--dereference	следовать символьным ссылкам (по умолчанию)
  -h	--no-dereference не следовать симвльным ссылкам
  -p	--preserve-date	не менять время доступа к файлам
  -s	--special-files	обрабатывать сециальные файлы (блочные/символьные устройства) как обычные

  -b	--brief		не выводит имя файла в своем выводе
  -n	--no-buffer	не буферизовать вывод
  -N	--no-pad	do not pad output
  -0	--print0	завершать имена файлов ASCII NUL'ом
  -r	--raw		не транслировать непечатаемые символы в \ooo

  -e	--exclude TEST	исключает TEST из списка тестов, которые применяются к файлу. 
			Возможны следующие типы тестов: apptype, ascii, cdf, compress, elf, encoding, soft, tar, text, tokens
  -i	--mime		выводит MIME тип (--mime-type and --mime-encoding)
	--apple		выводит apple CREATOR/TYPE
	--mime-type	выводит MIME type
	--mime-encoding	выводит MIME encoding
  -k	--keep-going	не останавливаться на первом совпадении

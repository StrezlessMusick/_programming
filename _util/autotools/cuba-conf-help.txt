`configure' configures Cuba 3.0 to adapt to many kinds of systems.

Usage: ./configure [OPTION]... [VAR=VALUE]...

Чтобы определить переменную среды (напр., CC, CFLAGS...), задайте ее как
VAR=VALUE.  Смотри ниже описание некоторых плезных переменных.

Значения по умолчанию для опций заданы в квадратных скобках

Конфигурация:
  -h, --help              показать эту справку и выйти
      --help=short        показать опции, специфичные для данного пакета
      --help=recursive    показать короткую справку всех включенных пакетов
  -V, --version           показать версию и выйти
  -q, --quiet, --silent   не печатать сообщения `checking ...'
      --cache-file=FILE   сохранить результаты тестов в FILE [disabled]
  -C, --config-cache      алиас для `--cache-file=config.cache'
  -n, --no-create         не создавать выходных файлов
      --srcdir=DIR        искать исходники в DIR [configure dir or `..'] ????

директории установки:
  --prefix=PREFIX         установить архитектурно-НЕзависимые файлы в PREFIX
                          [/usr/local]
  --exec-prefix=EPREFIX   установить архитектурно-зависимые файлы в EPREFIX
                          [PREFIX]

по умолчанию, `make install' будет устанавливать все файлы в
`/usr/local/bin', `/usr/local/lib' etc.  Вы можете задать
вы можете задать префикс установки отличный от `/usr/local' используя `--prefix',
к примеру `--prefix=$HOME'.

Для лучшего контроля используйте следующие опции:

>> Fine tuning - тонкая настройка
тонкая настройка директорий установки:
  --bindir=DIR            [EPREFIX/bin]		пользовательские бинарники
  --sbindir=DIR           [EPREFIX/sbin]	бинарники администрирования системы
  --libexecdir=DIR        [EPREFIX/libexec]	программные бинарники 
  --libdir=DIR            [EPREFIX/lib]		библитеки объектного кода
  --oldincludedir=DIR     [/usr/include]	C header files for non-gcc 
  --includedir=DIR        [PREFIX/include]	C header files 

  --sharedstatedir=DIR    [PREFIX/com]		модифицируемые системно-НЕзависимые данные
  --localstatedir=DIR     [PREFIX/var]		модифицируемые одномашинные данные
  --sysconfdir=DIR        [PREFIX/etc]		одно-машинные данные только для чтения
  --datarootdir=DIR       [PREFIX/share]	корень архитектурно-НЕзависимых данных только для чтения
  --datadir=DIR           [DATAROOTDIR]		архитектурно-НЕзависимые данные только для чтения
  --localedir=DIR         [DATAROOTDIR/locale]	локально-зависимые данные

  --infodir=DIR           [DATAROOTDIR/info]	info documentation 
  --mandir=DIR            [DATAROOTDIR/man]		man documentation 
  --docdir=DIR            [DATAROOTDIR/doc/cuba]documentation root 
  --htmldir=DIR           [DOCDIR]			html documentation 
  --dvidir=DIR            [DOCDIR]			dvi documentation 
  --pdfdir=DIR            [DOCDIR]			pdf documentation 
  --psdir=DIR             [DOCDIR]			ps documentation 

Дополнительные пакеты:
  --with-PACKAGE[=ARG]    использовать пакет PACKAGE [ARG=yes]
  --without-PACKAGE       не использовать PACKAGE (то же что --with-PACKAGE=no)
  --with-maxdim=N         максимальная размерность области интегрирования, если
                          массивы переменного размера не поддерживаются
  --with-maxcomp=N        максимальная размерность интегранда, если
                          массивы переменного размера не поддерживаются

>> influential - влиятельный
некоторые влиятельные переменные среды:
  MCFLAGS     флаги компилятора си MathLink
  CC          C compiler command
  CFLAGS      C compiler flags
  LDFLAGS     linker flags, e.g. -L<lib dir> if you have libraries in a
              nonstandard directory <lib dir>
  LIBS        libraries to pass to the linker, e.g. -l<library>
  CPPFLAGS    (Objective) C/C++ preprocessor flags, e.g. -I<include dir> if
              you have headers in a nonstandard directory <include dir>
  F77         Fortran 77 compiler command
  FFLAGS      Fortran 77 compiler flags

Use these variables to override the choices made by `configure' or to help
it to find libraries and programs with nonstandard names/locations.

PATCH(1)

ИМЯ
	patch - применяет файл изменений к оригинальному
СИНТАКСИС
	patch [опции] [исходный_файл [patch-файл]]
	
	но обычно
	
	patch -pчисло <patch-файл
ОПИСАНИЕ
	patch берет patch-файл содержащий список изменений произведенный программой diff и применяющий эти изменения к одному или более исходному файлу, создавая пропатченную версию. Обычно пропатченная версия кладется на место оригинальной. Может быть произведен возврат, см.  опции -b --backup. Имена файлов, которые будут пропатчены, обычно берутся из patch-файла, но если должен быть пропатчен только один файл, он может быть задан в командной строке как исходный_файл.

>> upon - на, при, по, после
>> attempt - пытаться
	При запуске patch пытается определить тип списка изменений, если он нен переопределен опцией -c (--context), -e (--ed), -n (--normal), или -u (--unified). (?)Context изменения (старый стиль, новый стиль, и unified) и normal изменения patch применяет самостоятельно, а ed изменения просто пересылает редактору ed через пайп (перенаправлением).

>> trail - тропа, идти по следу
>> trailing - задний
	(?)patch пытается пропустить мусор любого порядка, применить изменение, и пропустить любой следующий мусор. Таким образом вы можете послать любой документ или сообщение, содержащее список изменений для patch, и это должно работать. 
>> indent - отступ
>> intent - цель, намерение
>> intend - намереваться
>> consistant - последовательный
>> amount - количество, сумма
If the entire diff is indented by a consistent amount, если строки заканчиваются CRLF,  or if a diff is encapsulated one or more times by prepending "- " to lines starting with "-" as specified by Internet  RFC  934, this is taken into account.  After removing indenting or encapsulation, lines beginning with # are ignored, as they are considered to be comments.
>> consider - рассматривать
